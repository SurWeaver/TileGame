# Механики

Какие компоненты нужно добавлять на сущность для необходимого поведения.

## Общее

### DeltaPosition

Обновление позиции относительно родителя:

- Position (обновляется автоматически)
- DeltaPosition (смещение относительно родителя)
- ParentEntity
  - Position
  - \[Rotation\] (опционально, влияет на позицию дочерних элементов)

Если у родителя есть Rotation, позиция смещения поворачивается на угол родителя.

Сделано для того, чтобы прицеплять несколько спрайтов на одну сущность и двигать всё вместе.

### DeltaRotation

Обновление вращения относительно родителя:

- Rotation (обновляется автоматически)
- DeltaRotation (угол относительно родителя)
- ParentEntity
  - Rotation

Нужно, если объект с несколькими изображениями вращается, и дочерние изображения должны сохранять угол по отношению друг к другу.

### DeltaScale

Обновление масштаба относительно родителя:

- Scale (обновляется автоматически)
- DeltaScale (масштаб относительно родителя)
- ParentEntity
  - Scale

Для увеличения всех дочерних объектов относительно родителя.

### Родительские зависимости

Пока соблюдается, что каждый следующий дочерний элемент создан позже родителя, все позиции в иерархии обновляются последовательно, от родителя к дочерним элементам.

Если ссылка в `ParentEntity` становится недействительной, то вся сущность удаляется.
Если на сущности есть компонент `StayAfterParent`, то удаляется только компонент `ParentEntity`.

### Трюки с относительным позиционированием

Если на сущности следующие компоненты:

* Parent + **DeltaPosition** + Position + **DeltaRotation** + Rotation = объект двигается и вращается относительно родителя.
* Parent + **DeltaPosition** + Position + Rotation = объект двигается относительно родителя, сам вращается.
* Parent + Position + **DeltaRotation** + Rotation = объект вращается относительно родителя, стоит на своей позиции.

## Управление

### Ввод от пользователя

Сущность ввода:

- `PlayerAction`
- `ActionGamepadButton` / `ActionKeyboardKey` / `ActionMouseButton`
- \[`Happened`\]

Привязывается управление по взятому:

- `PlayerAction`
- `Happened(now: bool)` - только что или уже не первый кадр

### Нажатие UI-кнопок

Сущность кнопки:

- `ActionMouseButton`
- `ActionMouseRegion`
- \[`Happened`\]

Событие:

- `ActionMouseButton`
- `Happened`

```cs
EntityBuilder.NewEntity()
    .With(new ActionMouseButton(MouseButton.Left))
    .With(new ActionMouseRegion(new(0, 0, 32, 32)));
```

Срабатывает на отжатии.

Можно добавить на кнопку кастомное событие, и привязываться, например, так:

- `GameExitEvent`
- `Happened`

Или тот же `PlayerAction`, тогда будут срабатывать те же действия, что при вводе от пользователя.

## Визуал

### Удаление сущности после кадра

Нужно прицепить на сущность компонент `DeleteAfterFrameEnd`, чтобы вся сущность удалилась в конце кадра.

### Создание спрайтов

Достаточно создать на сущности:

- `Sprite(Texture)`
- `Position`
- `FillSpriteRequest`

Если на сущности будет `FillSpriteRequest` следующие поля заполнятся автоматически:
(Опускаю слово Sprite... в SpriteRectangle и т.д.)

- Rectangle (SourceRectangle)
- Color
- Rotation (Без Sprite..)
- Origin
- Scale
- Flip (SpriteEffects)
- Layer (Depth)

Без этих полей спрайт не будет отрисовываться.

До этого, если на сущности есть `AtlasTile` \[и `AtlasTileDelta`\], SourceRectangle заполнится автоматически, выделив фрагмент текстуры на "атласе".

До этого, если на сущности есть `SpriteAlignment`, то `Origin` вычислится от всей текстуры или от прикреплённого `AtlasTile`.

ВАЖНО! Для обновления `SpriteOrigin` или `SpriteRectangle`, достаточно удалить соответствующий компонент до работы `CalculateSprite\[Origin\Rectangle\]System`, чтобы в начале нового кадра он пересчитался.
